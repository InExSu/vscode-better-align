# План улучшения расширения

## Проблема

При Cmd+A (выделение всего кода) и нажатии Alt+A выравнивание не работает.

**Три сценария:**

| Сценарий | Действие | Результат |
|----------|----------|-----------|
| 1 | Курсор внутри кода, Alt+A | ✅ Выравнивает |
| 2 | Выделен участок кода, Alt+A | ✅ Выравнивает |
| 3 | Cmd+A (весь файл), Alt+A | ❌ Не выравнивает |

## Корень

Файл: `src/extension.ts:225`

```
analyzeSelection:
  если selection покрывает весь документ
    → fn_AutoSearchIndent(ctx)  ← ПРОБЛЕМА
```

`fn_AutoSearchIndent` (строка 260) определяет границы блока по **отступу активной строки**. При Cmd+A курсор может быть на строке с отступом `"  "`, и поиск захватывает только строки с таким же отступом:

```
{
  let x = 1         ← курсор здесь → отступ "  "
  let longName = 2  ← захватывается
  let veryLong = 3  ← захватывается
                    ← blank line → стоп
  if(x === 1) { }
  ...
}
```

Строки `{`, `if(...)`, `}` с другим отступом не попадают в rawLines.

## Решение 1 — Cmd+A отдаёт весь документ

В `analyzeSelection` заменить:

```typescript
if(selStart === 0 && selEnd === docLineCount - 1) {
    return fn_AutoSearchIndent(ctx) // было
}
```

на:

```typescript
if(selStart === 0 && selEnd === docLineCount - 1) {
    return { startLine: 0, endLine: docLineCount - 1 } // стало
}
```

При Cmd+A блоки определяются функцией `blocks_Find` (строка 389), которая группирует строки по пустым строкам/комментариям — корректно для любого кода независимо от отступа.

## Решение 2 — тест

Добавить в `test/align.test.ts` проверку с Cmd+A-подобным сценарием: передать все строки файла в `blocks_Find` без фильтрации по отступу и убедиться, что выравнивание работает.

## План работ

1. Исправить `analyzeSelection` в `src/extension.ts`
2. Написать тест: передача всех строк целиком → `blocks_Find` → выравнивание
3. `npm run test` + `npm run lint`
4. Собрать VSIX, установить, проверить в редакторе
5. Коммит

## Ожидаемый результат

После исправления Cmd+A + Alt+A выравнивает весь файл так же, как работают сценарии 1 и 2.
